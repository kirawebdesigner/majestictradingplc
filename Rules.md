# AI Coding Guardrails

## Style
- Use React function components and static data arrays for repeatable homepage sections.
- Keep component names descriptive.
- Keep styles in `src/styles.css` for this concept build.

## UX
- Preserve Strux-inspired interaction quality: reveal motion, tactile CTA hover, sticky nav, full-bleed imagery, and strong vertical rhythm.
- Make mobile navigation simple, large enough to tap, and keyboard accessible.

## Accessibility
- Use semantic sections, landmarks, labels, and alt text.
- Maintain visible focus states.
- Respect `prefers-reduced-motion`.

## Error Handling
- Static forms should prevent default submit and show non-destructive feedback only after a backend is added.

## Constraints
- Do not represent placeholder projects as confirmed real Majestic work.
- Clearly mark assumptions in docs and final notes.
